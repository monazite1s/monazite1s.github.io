递归函数

在函数内部调用自身的函数就是递归函数

比如计算阶乘：

使用递归函数时，每进入一次函数调用，栈就会增加一层栈帧，每当函数返回，栈就减少一层栈帧。当调用次数太多，就会导致栈溢出

尾递归优化

  • 为了避免栈溢出，可以通过尾递归进行优化。
  • 尾递归是指，在函数返回的时候，调用自身本身
  • 并且，return语句不能包含表达式

这样，编译器或者解释器就可以把尾递归做优化，使递归本身无论调用多少次，都只占用一个栈帧，不会出现栈溢出的情况。

遗憾的是，大多数编程语言没有针对尾递归做优化，Python解释器也没有做优化，所以，即使把上面的fact(n)函数改成尾递归方式，也会导致栈溢出。

私有变量

我们可以把数据封装在类中，通过定义属性和方法将抽象的逻辑封装在类内部。但是在定义一个类实例之后，外部代码仍然可以随意改变属性，为了让内部属性不被外部访问，py用下划线来规定了一个属性的访问限制：

  • 如score这样的属性可以自由的访问、修改

  • __name这样在属性名之前加上两个下划线表示将这个属性看作私有属性，外部代码不可以随意访问：

    • class Student(object)：
    def __init__(self,name,score):
        self.__name = name
        self.__score = score
        
    def print_score(self):
        print('%s: %s' % (self.__name, self.score))

    • 这种方法虽然比起通过外部代码直接修改属性而言更加繁琐，但是好处是可以对传入的参数进行判断，防止外部代码传入不符合规范的参数：

  • 在python中，有一类变量以__开头，又以__结尾。这种变量是特殊变量，其可以在外部代码直接访问，不是private变量

  • 有时候会遇到一个下划线开头的变量，如：_name这样的变量是可以在外部访问的。但是按照规范，这样的变量虽然可以被访问，但是我们应该把其视为私有变量，不能随意访问

原理

我们不能直接访问__name是因为python解释器将其改成了_Student__name。所以实际上类中并没有以__name命名的属性，而是变成了_Student__name。我们仍然可以直接访问_Student__name变量

  • 但是不同的Python解释器可能会把__name改成不同的变量
  • Python实际上并没有任何机制组织你干坏事，全靠自觉
